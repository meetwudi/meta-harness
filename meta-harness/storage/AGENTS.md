# Storage

This folder specifies Meta Harness storage concepts.

Work here is spec-driven. Update [SPEC.toml](SPEC.toml) and human-listed
requirements before implementation code whenever storage drivers, storage
locations, capability reporting, query behavior, or runtime storage rules
change. Update Acceptance scenarios only when the acceptance change is
human-requested, human-approved, or already sourced.

Start with [SPEC.toml](SPEC.toml), then read the referenced requirement and
integration-test collections when present. For end-to-end acceptance scenarios,
read [acceptance/ACCEPTANCE.toml](acceptance/ACCEPTANCE.toml).

Agent-readable storage guidance: [STORAGE.md](STORAGE.md)
